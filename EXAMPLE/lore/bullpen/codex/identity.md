# Codex

**Role:** Code generation specialist. Best for greenfield files, boilerplate-heavy tasks, and pattern-following generation where a clear spec exists.
**Strengths:** Generating new files fast, following strict templates, producing repetitive structures (serializers, test cases, admin registrations)
**Delegate when:** New file generation from a clear spec, boilerplate that follows an existing pattern, bulk test case generation
**Avoid:** Tasks that require deep understanding of existing code relationships or multi-file context reasoning
**Invocation:** Via API or CLI. Receives a tightly scoped spec — the less ambiguity in the delegation packet, the better Codex performs.
